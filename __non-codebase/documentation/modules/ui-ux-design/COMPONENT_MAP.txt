============================================================
🔧 ATOMIC UI COMPONENTS (building blocks):
============================================================

StatusBadge                    → Priority-colored status indicators
ActionButton                   → Context-aware action buttons  
├─ ActionButton.Critical       → Red buttons (DISPATCH, LOCKDOWN)
├─ ActionButton.Warning        → Orange buttons (BROADCAST, ESCALATE)  
├─ ActionButton.Success        → Green buttons (RESOLVE)
└─ ActionButton.Secondary      → Gray buttons (secondary actions)

GuardCard                      → Individual guard information
EvidenceViewer                 → Camera feed display
ProgressIndicator              → AI confidence metrics
├─ ProgressIndicator.Confidence → AI confidence percentage
└─ StatusLabel                 → Status indicators with variants

============================================================
🎛️ FILTERING & CONTROLS:
============================================================

EnterpriseActivityManager.tsx contains:
├─ Search Input                     → Text search across activities
├─ Priority Filters                 → [C] [H] [M] [L] buttons  
├─ Time Range Selector              → Live, 15m, 1h, 4h, 24h
├─ AI Filtering Toggle              → Smart false-positive filtering
├─ Clustering Toggle                → Group related activities
├─ View Mode Tabs                   → Minimal | Summary | Stream
└─ Layout Mode Toggle               → Grid | Horizontal scroll

============================================================
📊 DATA FLOW & INTEGRATION:
============================================================

DATA SOURCES:
useActivityStore() → Zustand store with activity data
useServices()      → Service layer for business logic
mockData          → Temporary test data (in components)

REAL-TIME UPDATES:
Activities Page → EnterpriseActivityManager → useActivityStore → Real data
Command Center  → Timeline → useIncidentStore → Real incidents  
Cases Page      → Cases → useCaseStore → Cases data (TO BUILD)

SERVICE INTEGRATION:
ActivityService    → Activity business logic
IncidentService    → Incident auto-creation  
CaseService        → Investigation management
AuditService       → Change tracking
